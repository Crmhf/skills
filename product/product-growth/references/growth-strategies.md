## 增长模型

### RICE 评分模型

```
RICE Score = Reach × Impact × Confidence / Effort

Reach: 每月/每季度触达用户数
Impact: 3=巨大, 2=高, 1=中, 0.5=低
Confidence: 100%=高, 80%=中, 50%=低
Effort: 人月投入
```

### ICE 快速评估

```
ICE = Impact × Confidence × Ease

用于快速筛选大量增长想法
```

## AARRR漏斗

### 各环节指标

| 阶段 | 核心指标 | 优化方向 |
|------|----------|----------|
| **Acquisition** | 新增用户、CAC | 渠道优化、降低获客成本 |
| **Activation** | 激活率、Aha时刻 |  onboarding优化 |
| **Retention** | 次日/7日/30日留存 | 产品价值、用户习惯 |
| **Revenue** | ARPU、LTV、转化率 | 付费转化、客单价提升 |
| **Referral** | 分享率、K因子 | 病毒传播、推荐激励 |

### 各阶段策略

**获客 (Acquisition)**
- SEO/ASO优化
- 内容营销
- 社交媒体
- 广告投放
- 合作推广

**激活 (Activation)**
- 简化注册流程
- 新手引导
- 快速体验核心价值
- 个性化推荐

**留存 (Retention)**
- 推送通知
- 邮件营销
- 用户分层运营
- 社区建设

## 增长实验

### 实验流程

```
1. 发现机会 → 数据分析、用户反馈、竞品研究
2. 提出假设 → 如果...那么...
3. 设计实验 → 实验组/对照组、样本量、周期
4. 执行实验 → 灰度发布、数据收集
5. 分析结果 → 统计显著性、业务影响
6. 决策落地 → 全量发布/迭代优化/放弃
```

### 实验模板

```markdown
## 实验名称：优化注册按钮颜色

### 假设
将注册按钮从蓝色改为橙色，可以提升点击率20%，
因为橙色更能吸引用户注意力。

### 实验设计
- 实验组：橙色按钮 (50%流量)
- 对照组：蓝色按钮 (50%流量)
- 指标：点击率、注册完成率
- 周期：2周
- 样本量：计算得出需10000用户

### 结果
- 点击率提升15% (p<0.05, 统计显著)
- 注册完成率无显著变化

### 结论
部分验证假设，建议全量发布并继续优化注册流程。
```

## 渠道策略

### 渠道分类

| 渠道类型 | 举例 | 特点 | 适用阶段 |
|----------|------|------|----------|
| 有机渠道 | SEO、内容、社媒 | 成本低、长期 | 全阶段 |
| 付费渠道 | 广告、SEM | 见效快、成本高 | 快速验证 |
| 病毒渠道 | 推荐、分享 | 成本低、难控制 | 成长期 |
| 商务渠道 | BD、合作 | 精准、门槛高 | 成熟期 |

### 渠道评估矩阵

```
            高转化率
               │
   重点投入     │     快速扩张
   (优化ROI)    │     (加预算)
               │
低 CAC ────────┼──────── 高 CAC
               │
   观察/放弃    │     测试优化
               │
            低转化率
```
