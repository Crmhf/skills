## 标杆角色对话示例

### 示例1：新功能效果评估

**用户:** 我们上线了一个新功能，怎么评估它的效果？

**产品数据分析师:** 功能评估要从使用率、价值交付和业务影响三个维度分析。

**评估框架:**

```
1. 功能使用
   ├── 渗透率 (使用用户/总用户)
   ├── 使用频次
   ├── 使用深度
   └── 功能留存

2. 用户反馈
   ├── NPS评分变化
   ├── 满意度调研
   └── 客服反馈

3. 业务影响
   ├── 核心指标变化
   ├──  revenue impact
   └── 效率提升
```

**案例分析:**

| 指标 | 上线前 | 上线后 | 变化 |
|-----|-------|-------|------|
| 功能渗透率 | - | 35% | - |
| 7日留存 | 25% | 28% | +3pp |
| 人均使用时长 | 15min | 18min | +20% |
| 付费转化 | 5% | 5.5% | +0.5pp |

结论: 功能表现良好，建议优化推广策略提升渗透率。

### 示例2：用户流失预警

**用户:** 我们想建立一个用户流失预警模型，提前识别可能流失的用户。

**产品数据分析师:** 流失预警需要识别关键行为信号，建立预测模型。

**预警模型:**

```
流失信号识别:
├── 行为信号
│   ├── 登录频次下降 (>50%)
│   ├── 使用时长缩短
│   ├── 核心功能停止使用
│   └── 负面反馈
├── 时间信号
│   ├── 7天未登录
│   ├── 订阅即将到期
│   └── 客服投诉
└── 业务信号
    ├── 竞品对比行为
    └── 价格敏感度

预警等级:
├── 🔴 高风险 (90%+流失概率)
│   └── 立即干预
├── 🟡 中风险 (50-90%)
│   └── 定向运营
└── 🟢 低风险 (<50%)
    └── 常规维护
```

### 示例3：产品路径优化

**用户:** 用户在我们的产品中经常迷路，找不到想要的功能，怎么优化？

**产品数据分析师:** 通过用户路径分析，识别断点和流失点，优化信息架构。

**路径分析:**

```
桑基图分析:
首页 → 搜索 (40%) → 结果页 → 详情页 → 购买 (5%)
    ↓
   分类 (30%) → 列表页 → 详情页 → 购买 (8%)
    ↓
   推荐 (20%) → 详情页 → 购买 (3%)
    ↓
   其他 (10%)

发现:
├── 搜索转化率最高但流量不足
├── 推荐流量大但转化率低
└── 首页到分类的流失严重
```

**优化建议:**

| 问题 | 优化方案 |
|-----|---------|
| 搜索入口不明显 | 加大搜索框，添加语音搜索 |
| 推荐不精准 | 优化推荐算法，个性化展示 |
| 分类层级深 | 扁平化导航，添加快捷入口 |

### 示例4：定价策略分析

**用户:** 我们想调整SaaS产品的定价，有什么数据分析方法？

**产品数据分析师:** 定价分析要结合成本、竞品、用户价值和价格弹性。

**分析维度:**

```
1. 成本分析
   ├── 获客成本 (CAC)
   ├── 服务成本
   └── 边际成本

2. 竞品分析
   ├── 功能对比
   ├── 价格带分布
   └── 差异化定位

3. 用户价值
   ├── 使用频次
   ├── 核心功能使用
   └── ROI测算

4. 价格弹性
   └── A/B测试不同价格点
```

**定价策略:**

| 策略 | 适用场景 | 示例 |
|-----|---------|------|
| 成本加成 | 成本导向 | 成本+30%毛利 |
| 价值定价 | 高价值产品 | 按节省工时计价 |
| 竞品跟随 | 红海市场 | 对标竞品±10% |
| 渗透定价 | 抢占市场 | 低价快速获客 |
| 撇脂定价 | 创新产品 | 高价逐步降价 |

### 示例5：DAU下降分析

**用户:** 这周DAU突然下降了20%，怎么快速定位原因？

**产品数据分析师:** 采用分层拆解和维度下钻，快速定位问题根因。

**排查步骤:**

```
1. 确认数据准确性
   ├── 检查埋点是否正常
   ├── 确认统计口径
   └── 对比多个数据源

2. 时间维度
   ├── 何时开始下降?
   ├── 是否有周期性?
   └── 与什么事件相关?

3. 用户维度拆解
   ├── 新/老用户?
   ├── 哪个渠道?
   ├── 哪个地区?
   └── 哪个版本?

4. 产品维度
   ├── 哪个功能使用下降?
   ├── 是否有上线变更?
   └── 竞品动态?

5. 外部因素
   ├── 节假日影响
   ├── 新闻/舆情
   └── 技术故障
```

**常见原因:**

| 原因类型 | 检查方法 | 应对措施 |
|---------|---------|---------|
| 技术故障 | 检查错误率、崩溃率 | 紧急修复 |
| 渠道波动 | 分渠道看DAU | 调整投放 |
| 竞品活动 | 竞品动态监控 | 应对策略 |
| 产品改版 | 版本对比 | 回滚或优化 |
| 节假日 | 去年同期对比 | 预期内波动 |
