## 用户故事版

适用场景：敏捷团队、以用户为中心的产品

```markdown
# 【产品名】用户故事集

## Epic（史诗）列表

| ID | Epic名称 | 描述 | 优先级 |
|----|----------|------|--------|
| E1 | 用户认证 | 注册、登录、找回密码 | P0 |
| E2 | 内容管理 | 创建、编辑、发布内容 | P0 |
| E3 | 社交互动 | 关注、点赞、评论 | P1 |

---

## 用户故事详情

### Epic: 用户认证 (E1)

#### Story 1: 用户注册

**故事描述：**
作为新用户，我希望通过手机号注册，以便快速开始使用产品。

**验收标准 (AC)：**
- [ ] 输入11位手机号，获取验证码
- [ ] 验证码6位数字，5分钟有效
- [ ] 验证通过后创建账号并登录
- [ ] 手机号已注册时提示"该手机号已注册，请直接登录"

**技术备注：**
- 接入阿里云短信服务
- 验证码存储Redis，TTL=300s

**估算：** 3故事点

---

#### Story 2: 用户登录

**故事描述：**
作为已注册用户，我希望通过手机号+密码登录，以便访问我的账号。

**验收标准：**
- [ ] 支持手机号+密码登录
- [ ] 支持手机号+验证码登录
- [ ] 密码错误3次后需输入图形验证码
- [ ] 登录成功后跳转至首页

**估算：** 2故事点

---

## 迭代规划

### Sprint 1 (2周)

| 故事 | 优先级 | 故事点 | 负责人 |
|------|--------|--------|--------|
| E1-S1 用户注册 | P0 | 3 | |
| E1-S2 用户登录 | P0 | 2 | |
| E2-S1 创建内容 | P0 | 5 | |

**容量：** 10故事点
```

## 技术对接版

适用场景：前后端对接、接口设计阶段

```markdown
# 【产品名】接口需求文档

## 接口列表

| 序号 | 接口名称 | 类型 | 优先级 | 依赖 |
|------|----------|------|--------|------|
| 1 | 用户注册 | POST | P0 | 无 |
| 2 | 用户登录 | POST | P0 | 无 |
| 3 | 获取用户信息 | GET | P0 | 用户登录 |

---

## 接口详情

### 1. 用户注册

**接口地址：** POST /api/v1/user/register

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | string | 是 | 手机号，11位 |
| code | string | 是 | 验证码，6位 |
| password | string | 是 | 密码，6-20位 |

**响应示例：**

```json
{
  "code": 200,
  "data": {
    "userId": "12345",
    "token": "eyJhbGciOiJIUzI1NiIs..."
  },
  "message": "注册成功"
}
```

**错误码：**

| 错误码 | 说明 |
|--------|------|
| 400001 | 手机号格式错误 |
| 400002 | 验证码错误或过期 |
| 400003 | 手机号已注册 |

**业务规则：**
1. 验证码通过短信发送，5分钟有效
2. 同一手机号每小时最多发送3次验证码
3. 注册成功后自动登录

## 状态定义

### 订单状态

```
待付款 -> 已付款 -> 待发货 -> 已发货 -> 已完成
   │        │         │         │         │
   └────────┴─────────┴─────────┴─────────┴──> 已取消
```

| 状态 | 说明 | 可转移状态 |
|------|------|------------|
| 待付款 | 订单创建未支付 | 已付款、已取消 |
| 已付款 | 支付成功 | 待发货、已取消 |
```
