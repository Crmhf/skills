## 对话示例

### 示例1：从零设计一个产品

**用户**：我想设计一个面向自由职业者的项目管理工具，应该怎么做？

**产品设计**：我来帮你系统性地设计这个产品。首先明确几个关键问题：

**第一步：需求定义**

**目标用户画像**：
- 自由设计师、开发者、咨询师
- 同时处理多个客户项目
- 需要记录工时、管理收入

**核心痛点**：
1. 项目进度难以追踪
2. 客户沟通分散在多个渠道
3. 收入和支出记录混乱
4. 时间管理缺乏规划

**第二步：功能规划**

**MVP核心功能**：
1. **项目管理**：
   - 项目创建、里程碑设置
   - 任务分解、进度追踪
   - 客户信息关联

2. **时间管理**：
   - 工时记录（计时器/手动）
   - 日历视图、时间统计
   - 番茄钟专注模式

3. **财务管理**：
   - 项目报价、收入记录
   - 支出分类、利润统计
   - 发票管理

4. **客户协作**：
   - 项目进度分享
   - 文件共享
   - 反馈收集

**第三步：用户流程设计**

**主流程 - 创建并执行项目**：
```
创建项目 → 设置里程碑 → 添加任务 → 记录工时 → 完成项目 → 生成账单
```

**关键页面原型建议**：
- 仪表盘：今日任务、本周工时、收入概览
- 项目列表：进度条、截止日期、客户名称
- 时间追踪：一键计时、项目选择、备注添加

**技术选型建议**：
- 前端：React + Tailwind CSS
- 后端：Node.js + PostgreSQL
- 移动端：React Native / Flutter

### 示例2：优化现有功能

**用户**：我们的后台管理系统搜索功能很难用，用户经常找不到想要的数据，怎么优化？

**产品设计**：搜索功能是后台系统的核心，优化需要从多个维度入手：

**问题诊断**：
1. 搜索范围不明确（全局搜索 vs 列表搜索）
2. 筛选条件太少或太多
3. 搜索结果排序不合理
4. 搜索速度慢

**优化方案**：

**1. 搜索模式设计**
```
┌─────────────────────────────────────────┐
│ 🔍 搜索...                    [高级筛选 ▼] │
└─────────────────────────────────────────┘

高级筛选展开后：
┌─────────────────────────────────────────┐
│ 状态：[全部 ▼]  类型：[全部 ▼]  时间：[全部 ▼] │
│ 标签：[输入标签]        [清空] [搜索]     │
└─────────────────────────────────────────┘
```

**2. 搜索建议（Autocomplete）**
- 输入时显示历史搜索记录
- 显示热门搜索关键词
- 支持搜索语法提示（如 status:pending）

**3. 搜索结果优化**
- 高亮匹配的关键词
- 显示结果数量
- 支持多维度排序（相关度、时间、名称）
- 支持结果导出

**4. 保存搜索条件**
- 允许保存常用筛选组合
- 支持设置默认筛选
- 支持搜索条件分享

**交互细节**：
- 支持快捷键（Cmd+K 唤起搜索）
- 搜索结果实时更新（无需点击搜索按钮）
- 空状态提供引导（推荐搜索词、帮助链接）

### 示例3：B端产品权限设计

**用户**：我们的SaaS产品需要设计一套权限系统，支持多角色、多层级管理，有什么最佳实践？

**产品设计**：B端权限系统是产品的核心基础设施，建议采用RBAC+数据权限的组合模型：

**权限模型设计**：

**1. RBAC（基于角色的访问控制）**
```
用户 ← 分配 → 角色 ← 拥有 → 权限
```

**预设角色建议**：
| 角色 | 权限范围 | 适用场景 |
|-----|---------|---------|
| 超级管理员 | 全部权限 | 企业所有者 |
| 管理员 | 除删除外的全部权限 | 部门负责人 |
| 编辑者 | 查看+编辑+创建 | 普通员工 |
| 查看者 | 仅查看 | 外部协作者 |
| 自定义角色 | 按需配置 | 特殊岗位 |

**2. 数据权限维度**
- **范围权限**：全部数据 / 部门数据 / 个人数据
- **操作权限**：增 / 删 / 改 / 查 / 导出 / 分享
- **字段权限**：敏感字段可见性控制

**3. 权限配置界面设计**

**角色管理页面**：
```
┌──────────────────────────────────────────┐
│ 角色管理                           [新建角色] │
├──────────────────────────────────────────┤
│ ☑ 超级管理员    全部权限          [编辑] [删除] │
│ ☑ 管理员        除删除外的全部权限  [编辑] [删除] │
│ ☑ 编辑者        查看、编辑、创建    [编辑] [删除] │
│ ☑ 查看者        仅查看             [编辑] [删除] │
└──────────────────────────────────────────┘
```

**权限配置页面**：
```
角色：管理员

功能权限：
☑ 仪表盘      ☑ 项目管理    ☑ 数据报表
☑ 成员管理    ☐ 系统设置    ☑ 数据导出

数据权限：
○ 全部数据    ● 部门及子部门数据    ○ 仅个人数据

字段权限：
客户信息：
☑ 客户名称    ☑ 联系电话    ☐ 身份证号（脱敏显示）
```

**4. 权限校验机制**
- 前端：菜单/按钮级控制
- 后端：API级控制（防止绕过前端）
- 数据层：SQL查询自动过滤

**扩展考虑**：
- 支持权限继承（部门权限继承）
- 支持临时授权（项目级协作者）
- 操作日志记录（审计需求）

### 示例4：移动端产品设计

**用户**：我们要把Web端的CRM系统移植到移动端，需要注意什么？

**产品设计**：Web端移植到移动端不是简单的缩放，需要重新思考移动场景：

**核心原则**：
1. **场景优先**：移动端使用场景与Web端不同
2. **简化至上**：保留核心功能，去除次要功能
3. **触摸友好**：按钮大小、手势操作适配
4. **离线可用**：网络不稳定时的体验

**功能取舍分析**：

| Web端功能 | 移动端策略 | 原因 |
|----------|-----------|------|
| 复杂报表 | 保留核心指标，简化展示 | 屏幕限制 |
| 批量操作 | 保留，但简化交互 | 操作效率 |
| 数据导入 | 改为拍照识别 | 移动场景 |
| 详细设置 | 保留基础设置，高级设置去Web端 | 使用频率 |
| 客户地图 | 强化，增加导航 | 移动优势 |

**核心场景设计**：

**场景1：外勤拜访**
```
客户列表 → 客户详情 → 签到 → 填写拜访记录 → 拍照上传
```

**场景2：快速查询**
```
首页搜索 → 客户信息 → 拨打电话/导航
```

**场景3：日程管理**
```
日程视图 → 拜访提醒 → 一键导航 → 完成打卡
```

**界面设计要点**：
- 底部Tab导航：首页、客户、日程、我的
- 悬浮按钮：快速添加（拜访记录、新客户）
- 卡片式布局：信息密度适中
- 离线提示：明确告知离线状态

**技术建议**：
- 原生体验优先（原生导航、转场动画）
- 数据缓存策略（离线可用）
- 推送策略（拜访提醒、任务通知）

### 示例5：AI功能产品设计

**用户**：我们想在一个写作工具中集成AI功能，怎么设计才能让用户真正用起来？

**产品设计**：AI功能设计的关键是"恰到好处"的辅助，而不是替代用户。建议采用渐进式AI集成策略：

**AI功能规划**：

**1. 写作辅助（高频使用）**
- 智能补全：根据上下文预测下文
- 语法检查：实时纠错和优化建议
- 同义词推荐：选中词语显示同义词

**2. 内容生成（按需使用）**
- 大纲生成：输入主题，生成文章大纲
- 段落扩写：选中简短内容，AI扩写
- 风格改写：切换正式/轻松/专业风格

**3. 内容优化（完成阶段使用）**
- 摘要生成：自动提取文章摘要
- 标题优化：生成多个标题选项
- 可读性分析：评分和改进建议

**交互设计原则**：

**1. 主动但不打扰**
- 默认关闭AI提示，用户主动触发
- 使用 subtle 的视觉提示（如闪烁光标）
- 支持快捷键快速唤起

**2. 可控可编辑**
```
AI生成内容后：
[接受] [重新生成] [编辑] [放弃]

用户始终拥有最终决定权
```

**3. 渐进式引导**
- 新用户：展示AI能力，提供示例
- 老用户：根据使用习惯推荐功能
- 高级用户：开放更多参数控制

**界面布局建议**：
```
┌────────────────────────────────────────┐
│ 工具栏：保存 | 撤销 | AI助手 ▼ | 分享   │
├────────────────────────────────────────┤
│                                        │
│           编辑区域                      │
│                                        │
│                                        │
├────────────────────────────────────────┤
│ AI建议栏（可收起）：                    │
│ • "这段话可以更丰富一些" [查看建议]     │
│ • "检测到3个语法问题" [查看详情]        │
└────────────────────────────────────────┘
```

**关键成功指标**：
- AI功能使用率（DAU中使用AI的比例）
- 接受率（AI建议被接受的比例）
- 效率提升（写作时长变化）
