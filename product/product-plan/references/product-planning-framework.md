## 产品生命周期

### 各阶段核心任务

| 阶段 | 目标 | 关键活动 | 成功指标 |
|------|------|----------|----------|
| **探索期** | 验证问题 | 用户访谈、竞品分析、MVP验证 | 问题-方案匹配度 |
| **验证期** | 验证产品 | 早期用户获取、迭代优化 | 产品-市场匹配度 |
| **成长期** | 快速扩张 | 规模化获客、功能完善 | 用户增长率 |
| **成熟期** | 稳定盈利 | 效率优化、变现提升 | 营收/利润率 |
| **衰退期** | 价值延续 | 成本控制、新方向探索 | ROI |

## 需求收集方法

### 方法对比

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 用户访谈 | 深度了解需求 | 信息丰富、可追问 | 样本量小 |
| 问卷调查 | 量化验证 | 样本量大、成本低 | 无法深入 |
| 数据分析 | 发现机会 | 客观、可量化 | 缺乏原因 |
| 竞品分析 | 对标学习 | 快速借鉴 | 可能不适用 |
| 内部反馈 | 一线洞察 | 直接、及时 | 可能片面 |

```
## 访谈前准备

## 核心问题框架
1. 背景：您的工作/生活中遇到过什么问题？
2. 现状：您目前是如何解决的？
3. 痛点：现有方案的不足之处？
4. 期望：理想方案是什么样的？
5. 价值：愿意为此付出什么？
```

## 优先级评估

### RICE 评分模型

```
RICE Score = (Reach × Impact × Confidence) / Effort

Reach: 影响用户数/周期
  - 例：每季度 5000 用户

Impact: 影响程度
  - 3 = 巨大影响
  - 2 = 高影响
  - 1 = 中等影响
  - 0.5 = 低影响

Confidence: 信心指数
  - 100% = 高信心
  - 80% = 中等
  - 50% = 低

Effort: 人月投入
  - 例：2 人月 = 2
```

### 优先级矩阵

```
            高价值
              │
    快速获胜   │   重点投入
    (P1)       │   (P0)
              │
低 effort ────┼────────── 高 effort
              │
    低优先级   │   需要论证
    (P3)       │   (P2)
              │
            低价值
```

## 路线图制定

### 路线图模板

```
## 产品路线图 2024

### Q1 目标：核心功能完善
- 1月：用户系统重构
- 2月：支付功能上线
- 3月：数据分析模块

### Q2 目标：体验优化
- 4月：性能优化专项
- 5月：移动端适配
- 6月：运营工具迭代

### Q3 目标：规模扩张
- 7月：国际化支持
- 8月：API 开放平台
- 9月：企业版功能

### Q4 目标：商业变现
- 10月：会员体系
- 11月：增值服务
- 12月：年度复盘
```

| 时间 | 主题 | 关键里程碑 | 负责人 | 状态 |

| Q1 | 基础功能 | 用户系统上线 | 张三 | 进行中 |
| Q2 | 体验优化 | 性能提升 50% | 李四 | 规划中 |
