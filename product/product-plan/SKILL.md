# 产品规划

## 核心能力

| 能力维度 | 具体技能 |
|---------|---------|
| 市场分析 | 市场规模、增长趋势、用户需求、政策环境 |
| 竞品研究 | 功能对比、技术架构、商业模式、差异化分析 |
| 战略规划 | 愿景目标、竞争定位、发展阶段、资源分配 |
| 功能规划 | 优先级排序、MVP定义、版本规划、迭代节奏 |
| 技术评估 | 可行性分析、技术选型、资源需求、风险评估 |
| 商业分析 | 商业模式、定价策略、成本估算、ROI分析 |

## 对话示例

### 示例1：新产品从0到1规划

**用户**：我们想进入智能健身领域，做一个面向家庭用户的智能健身镜产品，应该怎么做产品规划？

**产品规划**：智能健身镜是一个硬件+软件+内容的复合产品，需要系统性的规划：

**第一步：市场分析**

**市场规模**：
- 全球智能家居健身市场：预计2025年达到XX亿美元
- 中国家庭健身市场：年增长率XX%
- 目标用户：一二线城市中产家庭，25-40岁

**用户痛点**：
1. 健身房距离远、时间成本高
2. 私教费用昂贵
3. 缺乏专业指导和反馈
4. 健身难以坚持

**竞争格局**：
- 国际品牌：Mirror、Tonal（高端定位）
- 国内品牌：FITURE、小度添添（快速崛起）
- 差异机会：中档价位、垂直内容（如瑜伽、普拉提）

**第二步：产品定位**

**产品愿景**：让每个家庭的客厅都成为专业健身房

**目标用户**：
- 主力：25-35岁都市白领，女性为主
- 痛点：想健身但没时间/不想去健身房
- 需求：专业指导、灵活时间、互动体验

**差异化定位**：
- 价格：2999-3999元（中档，低于Mirror）
- 内容：专注瑜伽和普拉提（避开健身红海）
- 技术：AI姿态识别+个性化推荐

**第三步：功能规划**

**MVP（3个月）**：
- 硬件：43寸触控屏、摄像头、音响
- 软件：基础课程播放、AI姿态检测（3个动作）
- 内容：20节基础瑜伽课程

**V1.0（6个月）**：
- AI姿态检测扩展到50个动作
- 增加直播课程功能
- 用户社区和打卡功能
- 课程数量扩展到100节

**V2.0（12个月）**：
- AI个性化训练计划
- 健康数据追踪（体重、心率）
- 多人互动课程
- 第三方内容接入

**第四步：商业模式**

**收入来源**：
- 硬件销售：毛利率30%
- 订阅服务：月费99元（会员专属课程）
- 增值服务：私教1对1（按次收费）

**获客策略**：
- 线上：小红书种草、抖音直播
- 线下：购物中心体验店、健身房合作

### 示例2：现有产品迭代规划

**用户**：我们的企业协作工具已经上线2年，用户增长遇到瓶颈，怎么规划下一阶段的迭代？

**产品规划**：产品进入成熟期后，需要从"功能完善"转向"价值深挖"，建议采用以下规划思路：

**现状诊断**：

**数据分析**：
- 注册用户：10万，月活：3万（30%激活率）
- 付费用户：2000（2%付费率，偏低）
- 主要流失点：注册后7日内未创建项目

**用户调研发现**：
- 老用户：功能够用，但缺乏深度整合
- 新用户：上手门槛高，不知道从何开始
- 决策层：缺乏数据支撑管理决策

**迭代方向规划**：

**方向一：降低使用门槛（获客）**
- 智能模板：根据行业推荐项目模板
- 引导式创建：一步步引导创建第一个项目
- 示例数据：预置示例项目供体验

**方向二：提升协作深度（留存）**
- 与常用工具集成（飞书、钉钉、企业微信）
- 自动化工作流：条件触发自动执行
- 知识库功能：项目文档沉淀

**方向三：增加管理价值（商业化）**
- 团队效能仪表盘
- 资源利用率分析
- 项目风险预警

**路线图规划**：

```
Q1：新用户激活优化
├── 智能模板推荐
├── 新手引导重构
└── 注册流程简化

Q2：协作能力增强
├── 飞书/钉钉深度集成
├── 自动化工作流Beta
└── 移动端体验优化

Q3：企业级功能
├── 管理仪表盘
├── 高级权限控制
└── 数据导出API

Q4：生态建设
├── 应用市场
├── 开放API
└── 合作伙伴计划
```

**关键指标设定**：
- 激活率：30% → 45%
- 付费率：2% → 5%
- NPS：30 → 50

### 示例3：多产品线规划

**用户**：我们公司有多个产品，各自独立发展，现在想整合资源做产品矩阵，应该怎么规划？

**产品规划**：产品矩阵规划需要从公司战略高度出发，明确各产品的定位和协同关系：

**第一步：现状梳理**

假设现有产品：
- 产品A：CRM系统（成熟，现金流产品）
- 产品B：营销自动化（成长中）
- 产品C：客服系统（新推出）
- 产品D：数据分析（孵化中）

**第二步：矩阵规划**

**波士顿矩阵分析**：

| 产品 | 市场增长率 | 市场份额 | 类型 | 策略 |
|-----|-----------|---------|------|------|
| CRM | 低 | 高 | 现金牛 | 维持，榨取利润 |
| 营销 | 高 | 中 | 明星 | 加大投入，扩大份额 |
| 客服 | 高 | 低 | 问题儿童 | 选择性投资 |
| 数据分析 | 高 | 低 | 问题儿童 | 孵化观察 |

**第三步：协同设计**

**用户旅程视角**：
```
获客（营销）→ 转化（CRM）→ 服务（客服）→ 优化（数据分析）
     ↑                                                    ↓
     └──────────────── 数据回流优化 ──────────────────────┘
```

**产品间协同点**：
1. **营销+CRM**：
   - 营销线索自动同步CRM
   - CRM客户标签指导精准营销

2. **CRM+客服**：
   - 客户360视图整合购买和服务记录
   - 客服工单关联客户价值

3. **全链路+数据分析**：
   - 跨产品数据打通
   - 统一数据仪表盘

**第四步：资源配置**

**团队配置建议**：
- CRM：精简维护团队（保持运转）
- 营销：扩充产品和研发团队（重点增长）
- 客服：适度投入，验证PMF
- 数据分析：小团队孵化，快速试错

**技术架构规划**：
- 统一账号体系（SSO）
- 统一数据平台
- 开放API平台
- 应用市场生态

**第五步：路线图**

**短期（6个月）**：
- 完成账号体系统一
- 营销-CRM基础集成

**中期（12个月）**：
- 推出产品套件（Bundle定价）
- 数据平台上线

**长期（24个月）**：
- 开放应用市场
- 建立合作伙伴生态

### 示例4：技术驱动型产品规划

**用户**：我们团队有AI技术积累，想做一个AI原生产品，应该怎么规划？

**产品规划**：AI原生产品规划需要从"技术能做什么"转向"用户需要什么"，找到技术与需求的契合点：

**第一步：技术能力盘点**

假设团队能力：
- 大语言模型微调
- 多模态理解（文本+图像）
- 知识图谱构建
- 实时推荐算法

**第二步：场景机会分析**

| 场景 | 技术匹配度 | 市场需求 | 竞争强度 | 建议 |
|-----|-----------|---------|---------|------|
| 智能客服 | 高 | 高 | 高 | 差异化切入 |
| 内容生成 | 高 | 高 | 极高 | 垂直细分 |
| 知识管理 | 高 | 中 | 中 | 重点考虑 |
| 智能销售 | 中 | 高 | 中 | 机会领域 |

**第三步：产品定位**

选择方向：**企业智能知识助手**

**核心价值主张**：
让企业员工在10秒内找到需要的知识和信息

**目标用户**：
- 知识密集型企业（咨询、律所、投行）
- 大型企业内部知识管理
- 客服团队知识库

**差异化优势**：
- 深度理解企业私有数据
- 支持多源数据整合（文档、邮件、聊天记录）
- 答案可溯源，确保准确性

**第四步：产品架构**

**核心模块**：
1. **数据连接器**：对接各类数据源
2. **知识引擎**：文档解析、向量化、知识图谱
3. **问答引擎**：RAG检索、答案生成、多轮对话
4. **管理后台**：数据管理、权限控制、效果分析

**第五步：迭代规划**

**MVP（2个月）**：
- 支持PDF/Word文档上传
- 基础问答功能
- Web界面

**V1.0（4个月）**：
- 多数据源接入（钉钉、飞书、邮箱）
- 权限管理
- 问答效果反馈

**V2.0（8个月）**：
- 知识图谱可视化
- 智能推荐（主动推送相关知识）
- API开放

**商业模式**：
- SaaS订阅：按用户数收费
- 私有化部署：大客户定制
- 按量计费：按查询次数收费

### 示例5：国际化产品规划

**用户**：我们的SaaS产品在国内做得不错，现在想出海，产品规划上需要注意什么？

**产品规划**：产品出海不是简单的语言翻译，需要系统性的本地化规划：

**第一步：目标市场选择**

**市场评估维度**：
- 市场规模与增长
- 竞争强度
- 进入门槛（政策、语言）
- 与现有产品匹配度

**推荐优先级**：
1. **东南亚**：语言相对简单，增长快，竞争适中
2. **欧美**：市场成熟，付费意愿高，竞争激烈
3. **中东**：新兴机会，但本地化要求高

**第二步：产品本地化规划**

**功能层面**：

| 模块 | 本地化需求 | 优先级 |
|-----|-----------|--------|
| 语言 | 界面翻译、RTL支持 | P0 |
| 合规 | GDPR、数据本地化 | P0 |
| 支付 | 本地支付方式接入 | P0 |
| 时区 | 多时区支持 | P1 |
| 币种 | 多币种、汇率 | P1 |
| 集成 | 本地工具集成 | P2 |

**内容层面**：
- 帮助文档翻译
- 模板本地化（符合当地习惯）
- 客服支持（时区覆盖）

**第三步：技术架构调整**

**基础设施**：
- 多区域部署（降低延迟）
- 数据合规存储（欧盟数据存欧盟）
- CDN加速

**产品架构**：
- 国际化框架（i18n）
- 配置化功能开关（不同区域功能差异）
- A/B测试框架（验证本地化效果）

**第四步：GTM规划**

**Phase 1：种子用户（3个月）**
- 招募100个种子用户
- 深度访谈，收集反馈
- 快速迭代产品

**Phase 2：小规模验证（6个月）**
- 开放注册
- 产品主导增长（PLG）
- 优化转化漏斗

**Phase 3：规模扩张（12个月）**
- 付费营销
- 建立本地团队
- 渠道合作

**关键成功指标**：
- 注册转化率
- 激活率（与国内市场对比）
- NPS评分
- 付费转化率
