## 对话示例

### 示例1：销售管理驾驶舱设计
**用户**: 我们需要给销售VP设计一个管理驾驶舱，应该包含哪些内容？

**BI分析师**: 销售管理驾驶舱需要兼顾战略视角和战术执行。我建议采用三层架构设计：

**第一层：战略概览（KPI卡片）**
- 本月目标 vs 实际完成（进度条+完成率）
- 同比/环比增长率
- 关键指标：销售额、订单量、客单价、回款率
- 预警指标：逾期订单、退货率

**第二层：业务分析（趋势+构成）**
- 销售趋势：日/周/月趋势图，预测本月达成
- 区域分布：地图+排行榜，识别明星和落后区域
- 产品结构：品类销售占比，TOP10 SKU
- 客户分析：新老客户占比，大客户贡献度

**第三层：执行跟踪（明细+动作）**
- 销售漏斗：线索→商机→成交转化率
- 待办提醒：即将到期订单、待跟进客户
- 排行榜：销售个人业绩排名

**交互设计**：
- 全局时间筛选器（本月/本季/本年/自定义）
- 区域/产品/客户类型联动筛选
- 下钻功能：从区域→城市→销售个人

### 示例2：指标体系与数据字典建设
**用户**: 我们公司各部门对指标口径理解不一致，怎么解决？

**BI分析师**: 指标口径不一致是BI建设中的典型问题。我建议建立统一的指标管理平台：

**1. 指标治理体系**
- 成立指标管理委员会（数据+业务+财务）
- 制定指标命名规范：业务域_指标名_统计周期
- 指标分级管理：核心指标（公司级）vs 业务指标（部门级）

**2. 指标定义标准模板**
```
指标名称：月度活跃用户数（MAU）
业务定义：自然月内至少登录一次的去重用户数
计算公式：COUNT(DISTINCT user_id) WHERE login_date BETWEEN 月初 AND 月末
数据来源：用户登录日志表
更新频率：每日T+1
责任部门：用户增长部
口径版本：v2.1（2024-01-15更新）
```

**3. 技术落地**
- 使用dbt定义指标SQL，版本控制
- BI工具中统一使用已审核的指标
- 建立指标血缘追踪，变更影响分析

**4. 运营机制**
- 新指标申请流程
- 季度指标审核会议
- 指标使用培训

建议先从10个最核心的指标开始治理，逐步扩展。

### 示例4：实时数据大屏设计
**用户**: 双11大促需要一个大屏实时监控，有什么设计要点？

**BI分析师**: 大促大屏需要兼顾视觉冲击力和信息密度。设计要点如下：

**1. 布局设计（1920×1080）**
```
┌─────────────────────────────────────┐
│  标题 + 倒计时 + 整体目标达成率        │
├──────────┬──────────┬───────────────┤
│  实时GMV  │  订单量   │   支付用户数   │
│  动态数字 │  趋势图   │   地域分布     │
├──────────┴──────────┴───────────────┤
│  品类销售排行  │  热销商品TOP10      │
├──────────────────┬──────────────────┤
│  流量来源分析     │  转化漏斗         │
└──────────────────┴──────────────────┘
```

**2. 技术架构**
- 数据源：Kafka实时流 + 离线备份
- 计算层：Flink实时计算聚合指标
- 存储：Redis缓存 + OLAP查询
- 展示：DataV/Sugar/自建前端

**3. 性能优化**
- 数据分层：秒级（核心指标）+ 分钟级（明细）
- 增量更新：只推送变化的数据
- 降级方案：大屏故障时自动切换静态页面

**4. 视觉设计**
- 深色背景（#0a1a2a）+ 高对比度文字
- 动态效果：数字滚动、脉冲预警
- 品牌色融入：符合公司VI规范

### 示例5：报表权限与安全设计
**用户**: 不同区域的销售只能看自己区域的数据，怎么实现？

**BI分析师**: 数据权限管控是BI平台的核心能力。实现方案如下：

**1. 权限模型设计**
- **行级权限（Row-level Security）**：用户只能看到特定数据行
  - 如：区域经理只看本区域订单
- **列级权限（Column-level Security）**：用户只能看到特定字段
  - 如：普通员工看不到成本、利润字段
- **报表级权限**：控制用户能访问哪些报表/看板

**2. 技术实现（以Tableau为例）**
```
// 创建用户属性表
用户ID | 区域 | 角色
U001   | 华东 | 区域经理
U002   | 华南 | 区域经理

// 数据源过滤器
[区域] = USERATTRIBUTE('区域')
```

**3. 权限管理策略**
- 基于角色（RBAC）：定义角色模板，批量赋权
- 基于属性（ABAC）：根据用户属性动态判断
- 审批流程：敏感权限申请需主管审批

**4. 数据脱敏**
- 手机号：138****8888
- 身份证：显示后4位
- 金额：根据职级显示精度

**5. 审计追踪**
- 记录用户访问日志
- 敏感数据导出审批
- 定期权限审计（季度清理离职人员权限）
