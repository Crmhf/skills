### 示例1：SaaS产品UI设计规范

**用户:** 我们正在设计一个B端SaaS产品，需要建立一套专业的UI设计规范，有什么建议？

**UI设计师:** B端SaaS设计需要兼顾专业性和易用性。让我为您设计一套完整的设计规范。

**设计原则:**

| 原则 | 说明 | 应用 |
|-----|------|------|
| 效率优先 | 减少操作步骤，提高工作效率 | 快捷操作、批量处理 |
| 信息清晰 | 层级分明，重点突出 | 数据展示、表格设计 |
| 一致性 | 相同元素保持统一 | 交互模式、视觉样式 |
| 容错性 | 防止误操作，易于恢复 | 确认弹窗、撤销功能 |

**色彩系统:**

```
主色调:
├── Primary-50: #E6F0FF (浅色背景)
├── Primary-100: #B3D1FF
├── Primary-500: #2B7FFF (主色)
├── Primary-600: #1A6FEF (悬停)
└── Primary-700: #0D5FD9 (按下)

功能色:
├── Success: #00C853 (成功)
├── Warning: #FFB300 (警告)
├── Error: #FF1744 (错误)
└── Info: #00B0FF (信息)

中性色:
├── Gray-900: #1A1A1A (主文本)
├── Gray-600: #666666 (次要文本)
├── Gray-400: #999999 (禁用)
├── Gray-200: #E6E6E6 (边框)
└── Gray-100: #F5F5F5 (背景)
```

**字体规范:**

| 层级 | 字号 | 字重 | 行高 | 用途 |
|-----|------|------|------|------|
| H1 | 24px | 600 | 32px | 页面标题 |
| H2 | 20px | 600 | 28px | 区块标题 |
| H3 | 16px | 500 | 24px | 卡片标题 |
| Body | 14px | 400 | 22px | 正文内容 |
| Caption | 12px | 400 | 18px | 辅助说明 |

**间距系统 (8px基准):**

```
XS: 4px   - 图标内边距、紧凑间距
SM: 8px   - 组件内部间距
MD: 16px  - 卡片内边距
LG: 24px  - 区块间距
XL: 32px  - 模块间距
XXL: 48px - 页面间距
```

### 示例2：移动端App界面设计

**用户:** 我们要设计一个健身追踪App的UI，目标用户是25-35岁的都市白领，有什么设计建议？

**UI设计师:** 健身类App需要营造活力、专业的视觉感受，同时保持界面简洁易用。

**设计方向:**

**1. 品牌色彩**

```
活力橙 (Primary): #FF6B35
├── 代表: 能量、运动、激情
├── 使用: 主按钮、重点数据、进度指示
└── 搭配: 深色背景更醒目

深空灰 (Neutral): #1A1A2E
├── 代表: 专业、科技、沉稳
├── 使用: 背景、卡片、导航
└── 优势: OLED省电，护眼

薄荷绿 (Accent): #00D9C0
├── 代表: 健康、达成、清新
├── 使用: 成功状态、完成目标
└── 效果: 视觉焦点，平衡橙色
```

**2. 核心页面设计**

| 页面 | 设计要点 |
|-----|---------|
| 首页 | 今日数据可视化、快速开始训练入口 |
| 训练 | 大按钮设计、倒计时显示、心率实时图 |
| 统计 | 图表展示、周/月切换、个人记录 |
| 社区 | 卡片流、点赞交互、挑战活动 |
| 我的 | 成就徽章、设置入口、个人数据 |

**3. 组件设计示例**

```
今日步数卡片:
┌─────────────────────────────┐
│  👟 今日步数           ▶    │
│  ┌─────────────────────┐    │
│  │    8,432    /10,000 │    │
│  │    [████████░░░░░]  │    │
│  │    84% 已完成       │    │
│  └─────────────────────┘    │
│  🔥 消耗 342 千卡           │
└─────────────────────────────┘

样式:
- 卡片: 圆角 20px, 背景渐变
- 进度条: 高度 8px, 圆角 4px
- 数字: 字号 48px, 字重 Bold
```

**4. 交互细节**

- **动效**: 数据变化时数字滚动动画
- **反馈**: 按钮按下缩放0.95，提供触感反馈
- **过渡**: 页面切换使用共享元素过渡

### 示例3：设计系统构建

**用户:** 我们团队有多个产品，想构建一套统一的设计系统，从哪开始？

**UI设计师:** 设计系统建设是一个渐进过程，建议从底层Token开始，逐步向上构建。

**构建路线图:**

```
阶段1: 设计Token (2周)
  ├─ 颜色值定义
  ├─ 字体规范
  ├─ 间距体系
  └─ 阴影/圆角

阶段2: 基础组件 (4周)
  ├─ Button 按钮
  ├─ Input 输入框
  ├─ Select 选择器
  └─ Card 卡片

阶段3: 复合组件 (4周)
  ├─ Form 表单
  ├─ Table 表格
  ├─ Modal 弹窗
  └─ Navigation 导航

阶段4: 页面模板 (持续)
  ├─ 登录页
  ├─ 列表页
  ├─ 详情页
  └─ 仪表盘
```

**Design Token 设计:**

```json
{
  "color": {
    "primary": {
      "50": "#E6F0FF",
      "100": "#B3D1FF",
      "500": "#2B7FFF",
      "600": "#1A6FEF",
      "700": "#0D5FD9"
    },
    "neutral": {
      "white": "#FFFFFF",
      "100": "#F5F5F5",
      "900": "#1A1A1A"

  },

    "xs": "4px",
```

### 示例4：暗黑模式设计

**用户:** 我们的产品想支持暗黑模式，设计时需要注意什么？

**UI设计师:** 暗黑模式不是简单的颜色反转，需要重新设计色彩层级，确保可读性和舒适度。

**设计原则:**

| 原则 | 说明 |
|-----|------|
| 对比度 | 文字与背景对比度至少4.5:1 |
| 饱和度 | 降低色彩饱和度，避免刺眼 |
| 层级 | 通过亮度而非颜色区分层级 |
| 深度 | 越靠近用户的层越亮 |

**色彩映射:**

```
浅色模式 → 暗黑模式

Background:
  White (#FFF) → Gray-900 (#1A1A1A)

Surface:
  Gray-100 (#F5F5F5) → Gray-800 (#2D2D2D)

Elevated:
  White + Shadow → Gray-700 (#3D3D3D)

Text:
  Gray-900 → Gray-100
  Gray-600 → Gray-400
  Gray-400 → Gray-600

Primary:
  Primary-500 → Primary-400 (提高亮度)
```

**实现方案:**

```css
/* CSS Variables */
:root {
  --bg-primary: #FFFFFF;
  --bg-secondary: #F5F5F5;
  --text-primary: #1A1A1A;
  --text-secondary: #666666;
  --primary: #2B7FFF;
}

[data-theme="dark"] {
  --bg-primary: #1A1A1A;
  --bg-secondary: #2D2D2D;
  --text-primary: #F5F5F5;
  --text-secondary: #999999;
  --primary: #5B9FFF;
}
```

**Figma 变量设置:**

```
Variable Collections:
├── color/
│   ├── background/
│   │   ├── primary (Light: #FFF, Dark: #1A1A1A)
│   │   └── secondary (Light: #F5F5F5, Dark: #2D2D2D)
│   └── text/
│       ├── primary (Light: #1A1A1A, Dark: #F5F5F5)
│       └── secondary (Light: #666, Dark: #999)
└── spacing/
    ├── sm: 8px
    └── md: 16px
```

### 示例5：响应式设计

**用户:** 我们的Web产品需要适配桌面端、平板和手机，响应式设计有什么最佳实践？

**UI设计师:** 响应式设计的核心是内容优先，根据屏幕尺寸调整布局和交互方式。

**断点设计:**

| 断点 | 宽度 | 设备类型 | 布局策略 |
|-----|------|---------|---------|
| XS | < 576px | 手机竖屏 | 单列，全宽元素 |
| SM | 576-768px | 手机横屏/小平板 | 单列，部分双列 |
| MD | 768-992px | 平板 | 双列，侧边栏可折叠 |
| LG | 992-1200px | 小桌面/大平板 | 三列，完整导航 |
| XL | 1200-1400px | 桌面 | 多列，最大内容区 |
| XXL | > 1400px | 大桌面 | 居中，限制最大宽度 |

**布局模式演变:**

```
桌面端 (LG+):
┌─────────────────────────────────────────────┐
│  Logo    导航1 导航2 导航3    搜索   头像   │
├──────────┬────────────────────────────┬─────┤
│          │                            │     │
│  侧边栏   │        主内容区             │ 右栏 │
│          │                            │     │
│          │                            │     │
├──────────┴────────────────────────────┴─────┤
│                  页脚                       │
└─────────────────────────────────────────────┘

平板 (MD):
┌─────────────────────────────────┐
│  Logo   ☰    搜索        头像   │
├─────────────────────────────────┤
│                                 │
│           主内容区               │
│                                 │
├─────────────────────────────────┤
│           页脚                  │
└─────────────────────────────────┘

手机 (XS):
┌──────────────────┐
│  ☰  Logo    头像 │
├──────────────────┤
│                  │
│    主内容区       │
│    (单列堆叠)     │
│                  │
├──────────────────┤
│  ☆  ○  □  ○  ☰  │  ← 底部导航
└──────────────────┘
```

**组件响应式行为:**

```
数据表格:
- LG+: 完整表格，所有列显示
- MD:  关键列显示，次要列隐藏
- SM:  卡片式展示，横向滑动查看

导航栏:
- LG+: 水平导航，所有链接可见
- MD:  汉堡菜单，侧边抽屉
- XS:  底部Tab导航

图片:
- 使用 srcset 提供多尺寸
- 重要内容居中裁剪
- 响应式图片格式 (WebP)
```

**开发实现:**

```css
/* Container Queries (现代方式) */

  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}

/* 传统 Media Queries */
@media (max-width: 768px) {
  .sidebar {
    display: none;
  }
  .main-content {
    width: 100%;
  }
}
```
